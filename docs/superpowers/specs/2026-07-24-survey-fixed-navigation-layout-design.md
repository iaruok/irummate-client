# 설문 고정 내비게이션 레이아웃 디자인

## 목표

설문 본문이 화면보다 길어져도 진행 표시줄과 이전/다음 버튼을 항상 화면에 유지한다. 긴 본문은 두 고정 영역 사이에서만 세로로 스크롤되며 버튼 영역과 시각적으로 겹치지 않는다.

## 적용 범위

`MoveBtnGroup`을 사용하는 다음 페이지를 동일한 구조로 변경한다.

- `UserDetails`
- `SurveySleep`
- `SurveyClean`
- `SurveyLiving`
- `SurveyIntroduce`

## 레이아웃 구조

각 페이지의 최상위 `main`은 화면 높이인 `h-dvh`를 사용하고 `overflow-hidden`으로 페이지 전체 스크롤을 막는다. 내부는 다음 세 구역으로 구성한다.

1. 상단 고정 구역: `ProgressBar`
2. 중앙 스크롤 구역: 헤더, 설문 항목, 검증 오류
3. 하단 고정 구역: `MoveBtnGroup`

중앙 구역은 `min-h-0 flex-1 overflow-y-auto`를 사용한다. `min-h-0`은 flex 자식이 콘텐츠 높이 아래로 실제 축소될 수 있게 하며, 넘치는 내용은 중앙 구역 내부에서만 스크롤된다.

## 차폐와 간격

하단 버튼 구역은 `shrink-0`으로 높이를 유지하고 페이지 배경색을 가진다. 중앙 스크롤 구역의 하단에는 간격을 두어 마지막 입력 항목과 오류 메시지가 버튼 바로 아래에 가려지지 않고 완전히 노출될 수 있게 한다.

상단 ProgressBar와 하단 버튼은 스크롤 컨테이너 밖에 있으므로 별도의 `position: fixed` 없이도 화면에 계속 유지된다. 이 방식은 최대 너비 루트와 페이지 패딩 정렬을 그대로 보존한다.

## 동작과 접근성

설문 상태, 제출 동작, 검증 오류, 이동 경로, 키보드 포커스 동작은 변경하지 않는다. 스크롤 가능한 중앙 구역은 일반 문서 순서를 유지하므로 키보드 탐색 순서도 기존과 동일하다.

## 검증

- 정적 레이아웃 계약 테스트로 다섯 페이지가 화면 높이 제한, 고정 ProgressBar, 중앙 스크롤 구역, 스크롤 밖 `MoveBtnGroup` 구조를 공유하는지 확인한다.
- 긴 항목을 가진 `SurveyLiving`이 중앙 스크롤 구역을 사용하는지 별도로 확인한다.
- 전체 Node 테스트, ESLint, Vite 프로덕션 빌드를 실행한다.
