# 모달 테스트 대시보드 설계

## 목적

공개 `/test` 페이지에서 공통 `Modal` 컴포넌트의 콘텐츠 구성, 닫기 방식, 크기, 버튼 variant, 스크롤과 반응형 동작을 로컬 브라우저로 확인할 수 있게 한다.

## 범위

- `src/pages/Test.jsx`만 구현한다.
- 사용자가 이미 추가한 `/test` 라우트와 `src/App.jsx` 변경은 수정하지 않는다.
- 운영 기능이나 API를 호출하지 않는 개발 확인용 페이지로 구성한다.
- 별도 이미지 파일을 추가하지 않고 CSS 기반 예시 캡처 영역을 사용한다.

## 화면 구성

페이지 상단에는 제목과 간단한 안내를 표시한다. 그 아래에 다음 세 종류의 모달 실행 카드를 둔다.

1. 도움말 모달: 제목, 설명, 여러 문단의 본문과 단일 닫기 버튼
2. 이미지 예시 모달: `large` 크기, CSS로 만든 기숙사 합격 화면 예시, 닫기와 확인 버튼
3. 확인 모달: 취소, 확인 버튼 및 확인 결과 표시

공통 옵션 영역에는 다음 체크박스를 제공한다.

- 배경 클릭으로 닫기
- Esc 키로 닫기
- 우측 상단 X 버튼 표시

페이지 하단에는 마지막으로 발생한 닫기 사유 또는 액션 결과를 표시한다. 모달의 `onClose`가 받은 `overlay`, `escape`, `close-button`과 Footer의 `cancel`, `confirm`을 구분한다.

## 상태 흐름

`activeModal`은 `help`, `image`, `confirm`, `null` 중 하나다. 각 실행 버튼은 해당 값을 설정하고, 공통 `closeModal(result)` 함수는 결과 문구를 저장한 뒤 `activeModal`을 `null`로 만든다.

옵션 체크박스는 `closeOnOverlayClick`, `closeOnEscape`, `showCloseButton` 상태를 변경하며 세 모달에 동일하게 전달한다.

## 스타일 및 접근성

기존 Tailwind 토큰을 사용해 모바일 우선 카드형 대시보드를 구성한다. 실행 버튼과 체크박스에는 명시적인 텍스트 레이블을 제공한다. CSS 기반 예시 이미지는 시각적 장식이 아니라 확인 대상이므로 `role="img"`와 설명 가능한 `aria-label`을 제공한다.

긴 도움말 본문은 여러 문단과 목록을 포함해 모달 내부 스크롤 및 sticky Footer를 확인할 수 있게 한다.

## 검증

- `/test`에서 세 실행 버튼이 각각 올바른 모달을 연다.
- 옵션 체크박스가 배경, Esc, X 버튼 동작에 반영된다.
- 마지막 결과에 닫기 사유와 확인/취소 결과가 표시된다.
- 도움말의 긴 본문이 viewport를 벗어나지 않고 모달 내부에서 스크롤된다.
- 이미지 예시 모달이 모바일과 데스크톱에서 적절한 너비로 표시된다.
- ESLint와 Vite 프로덕션 빌드가 통과한다.
