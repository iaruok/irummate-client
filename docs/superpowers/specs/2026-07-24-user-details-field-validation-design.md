# UserDetails 필드 형식 검증 설계

## 목표

`UserDetails`의 모든 필수값이 입력된 상태에서 나이, 전화번호, 학번 형식이 올바르지 않으면 각 라벨 오른쪽에 지정된 오류 메시지를 표시하고 제출을 중단한다.

## 검증 우선순위

1. 다음 버튼 클릭 시 변환 전 원본 입력값의 존재 여부를 먼저 검사한다.
2. 하나라도 비어 있으면 기존 `RequiredFieldsModal`만 표시하고 필드 형식 오류는 모두 숨긴다.
3. 모든 필수값이 존재할 때만 필드 형식 검증을 실행한다.
4. 형식 오류가 있으면 status 및 상세 정보 API를 호출하지 않는다.
5. 오류가 없을 때만 나이를 숫자로 변환하고 기존 role 기반 POST/PATCH 흐름을 실행한다.

## 형식 규칙

- 나이: 숫자 변환 결과가 정수가 아니거나 0 이하이면 `나이가 올바르지 않습니다.`
- 전화번호: `^010-\d{4}-\d{4}$`와 일치하지 않으면 `전화번호 형식이 올바르지 않습니다.`
- 학번: 문자열 길이가 정확히 10이 아니면 `학번 형식이 올바르지 않습니다.`

## UI와 상태

- `InlineInput`에 선택적 `errorMessage` 속성을 추가한다.
- 오류는 라벨 오른쪽에 `text-fg-error text-xs font-bold` 스타일로 표시한다.
- 최초 입력 중에는 오류를 표시하지 않는다.
- 모든 필수값이 있는 상태에서 다음 버튼을 누른 뒤 형식 검증을 활성화한다.
- 검증 활성화 이후 나이, 전화번호, 학번을 수정하면 전체 필드 오류를 즉시 다시 계산한다.
- 사용자가 값을 비운 뒤 다음 버튼을 누르면 필드 오류를 숨기고 기존 필수 입력 모달을 표시한다.

## 테스트

- 누락 검사가 빈 문자열만 누락으로 판정하는지 검사한다.
- 세 필드의 유효·무효 경계를 순수 함수로 검사한다.
- 필수값 모달이 형식 오류보다 먼저 처리되는지 검사한다.
- 형식 오류가 status 및 제출 API보다 먼저 처리되는지 검사한다.
- InlineInput의 오류 메시지와 정확한 스타일을 검사한다.
- 집중 테스트, 린트, 프로덕션 빌드를 실행한다.
