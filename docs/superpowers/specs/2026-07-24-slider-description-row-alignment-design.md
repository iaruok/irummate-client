# 슬라이더 설명 라벨 높이 및 중심 정렬 설계

## 목표

취침 시간대 이외의 슬라이더에서 1번과 5번 아래 설명을 3번 아래 `보통`과 같은 높이에 배치하고, 각 설명의 가로 중심을 1·3·5 인덱스의 중심과 일치시킨다.

## 원인

현재 3번의 `보통`은 `indexLabels` 컨테이너에 렌더링되지만, 1번과 5번의 `leftDescription`·`rightDescription`은 그 아래 별도의 `mt-2` 컨테이너에 렌더링된다. 서로 다른 행을 사용하므로 1번과 5번 설명이 더 아래에 표시된다.

## 동작

- `leftDescription`은 1번 인덱스 위치에 표시한다.
- 기존 `indexLabels` 값은 각 인덱스 위치에 표시한다.
- `rightDescription`은 마지막 인덱스 위치에 표시한다.
- 모든 설명은 같은 `top-0` 높이를 사용한다.
- 각 설명은 해당 숫자 인덱스와 동일한 퍼센트 위치 및 `-translate-x-1/2`를 사용해 가운데 정렬한다.
- 취침 시간대의 두 줄 인덱스 라벨 동작은 유지한다.

## 구현

`Slider`에서 표시할 라벨을 `indexLabels`에 끝 설명을 합친 하나의 객체로 구성한다. `leftDescription`이 있으면 1번 값을, `rightDescription`이 있으면 마지막 번호 값을 추가한다. 기존 인덱스 라벨과 끝 설명이 동시에 같은 위치에 전달되는 경우 명시적인 `indexLabels` 값을 우선한다.

통합된 라벨 객체를 기존 절대 위치 라벨 렌더링에 사용하고, 별도의 끝 설명 행은 제거한다. 모든 라벨은 현재 인덱스 위치 계산과 가운데 정렬 클래스를 공유한다.

## 검증

- `Slider`가 끝 설명을 통합된 인덱스 라벨 객체에 포함하는지 확인한다.
- 1·3·5 라벨이 동일한 절대 위치 컨테이너와 가운데 정렬 클래스를 사용하는지 확인한다.
- 별도의 `mt-2` 끝 설명 행이 제거됐는지 확인한다.
- 취침 시간대 두 줄 라벨, 전체 테스트와 빌드가 유지되는지 확인한다.
