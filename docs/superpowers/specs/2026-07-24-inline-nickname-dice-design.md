# 인라인 닉네임 주사위 디자인

## 목표

닉네임 무작위 생성 버튼을 별도 흰색 버튼에서 닉네임 입력 영역의 오른쪽 끝으로 이동한다. 입력 텍스트와 버튼이 겹치지 않으며 하나의 입력 컨트롤처럼 보이게 한다.

## 컴포넌트 설계

기존 `InlineInput`에 선택적 `endAdornment` prop을 추가한다. 새 닉네임 전용 입력 컴포넌트는 만들지 않는다. 전용 컴포넌트는 라벨, ID 연결, 입력 이벤트, 상태 속성, 기본 스타일을 복제하므로 유지보수 비용이 더 크다.

`InlineInput`은 다음과 같이 변경한다.

- label 구조는 유지한다.
- input을 `relative` 래퍼로 감싼다.
- `endAdornment`가 있을 때만 input에 `pr-12`를 추가한다.
- 부가 요소는 입력 영역 오른쪽 끝에 수직 중앙 정렬한다.
- `endAdornment`가 없는 기존 사용처의 시각적 결과와 동작은 유지한다.

## 닉네임 영역

`SurveyIntroduce`는 기존의 입력/버튼 가로 배치를 제거하고 `InlineInput` 전체를 사용한다. 주사위 버튼을 `endAdornment`로 전달한다.

- 버튼은 입력 내부 오른쪽에 약 36px 크기의 클릭 영역을 가진다.
- 버튼 배경은 투명하며 hover 시 `ui-sub`를 사용한다.
- 기존 무작위 닉네임 생성 동작과 접근성 라벨을 유지한다.
- 입력 텍스트는 오른쪽 내부 여백으로 버튼 영역을 침범하지 않는다.

## 범위와 검증

닉네임 생성 로직과 다른 입력 필드의 동작은 변경하지 않는다. 정적 컴포넌트 계약 테스트로 선택적 adornment, 입력 오른쪽 여백, 버튼의 입력 내부 배치를 확인한다. 전체 테스트, ESLint, Vite 프로덕션 빌드를 실행한다.
