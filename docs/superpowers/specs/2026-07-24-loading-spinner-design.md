# 로딩 스피너 통합 설계

## 목표

페이지 이동, 데이터 조회, 폼 제출 등 실제 비동기 작업이 진행되는 동안 화면에 표시되는 로딩 문구를 일관된 스피너로 교체한다. 오류, 빈 상태, 업무 대기 상태를 설명하는 문구는 유지한다.

## 범위

다음과 같이 비동기 상태에 의해서만 임시로 표시되는 문구를 교체한다.

- 로그인 상태 확인
- 채팅방 목록 및 채팅방 초기 데이터 조회
- 이전 메시지 추가 조회
- 매칭 추천 목록 조회
- 마이페이지 데이터 조회
- 기존 설문 데이터 조회
- 관리자 회원, 인증 요청, 인증 상세, 매칭 설정 조회
- 인증 확인, 설문 제출·수정, 로그아웃, 회원 탈퇴, 관리자 제재, 매칭 거절, 연락처 확인 등 버튼 또는 부분 영역의 비동기 작업

다음 문구는 교체하지 않는다.

- 오류 및 재시도 안내
- 조회 결과가 없는 빈 상태 안내
- 상대방 확정 대기처럼 서버에 저장된 업무 상태
- 채팅 서버 연결 실패 또는 연결 대기 때문에 사용자가 행동할 수 없는 이유

## 컴포넌트

`src/components/LoadingSpinner.jsx`에 공통 컴포넌트를 둔다.

- `label`: 스크린 리더에 전달할 필수 상태 설명
- `size`: `sm`, `md`, `lg` 중 하나이며 기본값은 `md`
- `className`: 배치 영역에서 색상이나 여백을 조정할 때 사용

컴포넌트는 `role="status"`와 `aria-live="polite"`를 가진 컨테이너, CSS 테두리와 `animate-spin`으로 만든 시각적 원형 스피너, `sr-only` 상태 문구를 렌더링한다. SVG나 외부 패키지를 추가하지 않는다.

## 적용 방식

페이지와 섹션 로딩은 기존 컨테이너의 높이, 배경, 정렬을 유지하고 그 안의 텍스트 노드만 공통 스피너로 바꾼다. 따라서 데이터가 도착했을 때 발생하는 레이아웃 이동을 늘리지 않는다.

버튼 로딩은 기존 레이블 대신 작은 스피너를 표시한다. 버튼의 기존 `disabled` 상태를 유지하고, 비동기 상태 동안 업무별 `aria-label`을 제공한다. 가능한 경우 고정된 최소 너비를 유지해 스피너 전환 시 버튼 폭이 급격히 줄지 않게 한다.

연락처 확인처럼 로딩 문구와 정상 안내가 같은 위치를 공유하는 경우 로딩 중에는 스피너만 표시하고, 완료 후 기존 정상 안내를 표시한다.

이미 회전 아이콘을 사용하는 오늘의 매칭 실행 버튼은 기존 구현이 동일한 시각 패턴을 충족하므로 유지하되, 접근 가능한 상태 이름이 보존되는지 확인한다.

## 접근성

- 시각적 스피너는 장식 요소로 처리한다.
- 모든 스피너에는 구체적인 한국어 상태 설명을 `sr-only`로 제공한다.
- 페이지 및 섹션 로딩은 상태 변경을 보조 기술이 알 수 있도록 `role="status"`와 `aria-live="polite"`를 사용한다.
- 버튼은 로딩 중 비활성화하고 작업별 `aria-label`을 제공한다.
- `prefers-reduced-motion` 사용자를 위해 전역 CSS를 추가하지 않는다. Tailwind의 애니메이션 유틸리티가 생성하는 모션 감소 변형을 컴포넌트에 적용해 회전을 중단한다.

## 오류 처리와 상태 전환

기존 비동기 상태, `try/catch/finally`, 오류 문구 및 재시도 흐름은 변경하지 않는다. 이번 작업은 렌더링 표현만 바꾸며 네트워크 호출이나 상태 소유권을 이동하지 않는다.

## 검증

- 공통 스피너가 시각적 요소와 숨겨진 상태 설명을 렌더링하는지 테스트한다.
- 지원하는 크기별 클래스가 적용되는지 테스트한다.
- 소스 검색으로 실제 비동기 상태에서 렌더링하던 로딩 문구가 남아 있지 않은지 확인한다.
- 기존 테스트, ESLint, 프로덕션 빌드를 실행한다.
- 오류, 빈 상태, 업무 대기 안내 문구가 그대로 남아 있는지 검토한다.
