# UserDetails 필수 입력 안내 모달 설계

## 목표

`UserDetails` 페이지에서 필수 입력 항목이 누락된 채 다음 버튼을 누르면 설문 단계로 이동하지 않고, 공용 모달 본문에 `모든 항목은 필수 입력입니다.`를 표시한다.

## 동작

1. 다음 버튼 클릭 시 이름, 나이, 성별, 전화번호, 학번, 학과를 검사한다.
2. 문자열 값은 앞뒤 공백을 제거한 뒤 비어 있는지 확인하고, 나이는 숫자로 변환했을 때 0보다 큰지 확인한다.
3. 누락된 값이 있으면 API를 호출하지 않고 안내 모달을 연다.
4. 클라이언트 검증을 통과했더라도 사용자 정보 API가 HTTP 400을 반환하면 같은 안내 모달을 연다.
5. 다른 오류는 기존처럼 콘솔에 기록하며, 성공 시 현재 사용자 정보를 갱신하고 다음 설문으로 이동한다.

## UI

- 기존 공용 `Modal` 컴포넌트를 사용한다.
- 제목: `입력 확인`
- 본문: `모든 항목은 필수 입력입니다.`
- 푸터 버튼은 표시하지 않는다.
- 우측 상단 닫기 버튼, 배경 클릭, Escape 키로 닫을 수 있다.

## 테스트

- 필수 항목이 누락되면 API를 호출하지 않고 지정 문구의 모달을 표시한다.
- 모든 항목이 채워졌지만 API가 400을 반환하면 같은 모달을 표시한다.
- 유효한 입력과 성공 응답에서는 사용자 정보를 갱신하고 `/surveys/sleep`으로 이동한다.
- 변경 후 린트와 빌드를 실행한다.
