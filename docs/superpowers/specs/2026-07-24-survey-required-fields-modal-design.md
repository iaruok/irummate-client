# 설문 필수 입력 공용 모달 설계

## 목표

`SurveySleep`, `SurveyClean`, `SurveyLiving`에서 필수 응답을 누락한 채 다음 버튼을 누르면 저장 및 화면 이동을 중단하고, `UserDetails`와 동일한 필수 입력 안내 모달을 표시한다.

## 공용 컴포넌트

- `RequiredFieldsModal` 컴포넌트를 공용 컴포넌트 폴더에 만든다.
- `open`과 `onClose`만 외부에서 받는다.
- 내부에서 기존 공용 `Modal`을 사용한다.
- 제목은 `입력 확인`, 본문은 `모든 항목은 필수 입력입니다.`로 고정한다.
- 푸터 확인 버튼은 표시하지 않는다.
- 우측 상단 닫기 버튼, 배경 클릭, Escape 키로 닫을 수 있다.
- `UserDetails`도 이 컴포넌트를 사용하도록 변경해 네 페이지의 UI와 동작을 한 곳에서 관리한다.

## 설문 페이지 동작

1. 각 페이지는 현재 사용 중인 `Number.isInteger` 기반 필수 응답 검사를 유지한다.
2. 누락된 응답이 있으면 모달을 열고 즉시 반환한다.
3. 누락된 상태에서는 설문 초안을 추가 저장하거나 다음 페이지로 이동하지 않는다.
4. 모든 응답이 있으면 기존처럼 초안을 저장하고 다음 설문 페이지로 이동한다.
5. 기존 `errorMessage` 상태와 화면 하단 인라인 오류 문구는 제거한다.

## 테스트

- 공용 모달에 정확한 제목과 본문이 있고 푸터 버튼이 없는지 검사한다.
- 세 설문 페이지가 누락 검사 실패 시 공용 모달을 열도록 연결됐는지 검사한다.
- 세 설문 페이지에서 기존 인라인 오류 상태와 문구가 제거됐는지 검사한다.
- `UserDetails`가 공용 모달을 사용하도록 변경됐는지 검사한다.
- 기존 설문 레이아웃 테스트, 린트, 프로덕션 빌드를 실행한다.
