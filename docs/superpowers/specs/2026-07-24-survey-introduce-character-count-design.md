# SurveyIntroduce 글자 수 표시 설계

## 목표

`SurveyIntroduce`의 닉네임과 자기소개 입력 제한을 사용자가 입력 중에 알 수 있도록, 각 라벨 오른쪽에 `현재 글자 수/최대 글자 수` 카운터를 실시간 표시한다.

## 공용 입력 컴포넌트

- `InlineInput`과 `TextArea`에 선택적 `showCharacterCount` 속성을 추가한다.
- `showCharacterCount`가 참이고 `maxLength`가 숫자일 때만 카운터를 표시한다.
- 현재 글자 수는 `String(value ?? '').length`로 계산해 공백과 줄바꿈을 포함한다.
- 라벨 행은 좌우 정렬하며 기존 라벨 텍스트와 필수 표시를 왼쪽, 카운터를 오른쪽에 배치한다.
- 카운터는 작은 보조 텍스트 스타일을 사용한다.
- 카운터는 입력 제한을 설명하는 시각적 보조 정보이며 별도의 라이브 영역으로 반복 안내하지 않는다.

## SurveyIntroduce 적용

- 닉네임 `InlineInput`에 `showCharacterCount`를 전달해 `0/8`부터 표시한다.
- 자기소개 `TextArea`에 `showCharacterCount`를 전달해 `0/200`부터 표시한다.
- 자기소개 `maxLength`를 기존 `500`에서 실제 제출 검증 기준인 `200`으로 변경한다.
- 기존 닉네임 8자 및 자기소개 200자 제출 전 검증은 우회 입력과 비정상 저장 데이터를 대비해 유지한다.
- 다른 입력 컴포넌트 사용처에는 카운터를 표시하지 않는다.

## 테스트

- 두 공용 입력 컴포넌트가 선택적 카운터와 현재/최대 길이를 렌더링하는 구조인지 검사한다.
- `SurveyIntroduce`에서 닉네임과 자기소개 모두 카운터를 활성화했는지 검사한다.
- 자기소개 `maxLength`가 `200`인지 검사한다.
- 기존 집중 테스트, 린트, 프로덕션 빌드를 실행한다.
