# UserDetails 드롭다운 자동 스크롤 설계

## 목표

UserDetails 페이지의 드롭다운 메뉴를 열었을 때 목록 하단이 현재 뷰포트를 벗어나면, 전체 메뉴를 확인할 수 있도록 페이지를 필요한 거리만큼 부드럽게 아래로 이동한다.

## 동작

- 마우스 클릭과 키보드 조작으로 드롭다운을 여는 경우에 모두 적용한다.
- 메뉴가 DOM에 렌더링된 다음 실제 위치와 높이를 측정한다.
- 메뉴 하단과 뷰포트 하단 사이에 16px 여백을 확보한다.
- 메뉴 하단이 뷰포트를 벗어난 경우에만 넘친 거리만큼 페이지를 아래로 부드럽게 스크롤한다.
- 메뉴가 이미 뷰포트 안에 모두 보이면 페이지 위치를 변경하지 않는다.
- 기존의 활성 옵션을 목록 내부에서 보이게 하는 스크롤 동작은 유지한다.

## 구현

`DropDownMenu`에 목록 요소를 가리키는 ref를 추가한다. `isOpen`이 참으로 바뀌어 목록이 렌더링되면 `requestAnimationFrame`에서 `getBoundingClientRect()`를 호출해 메뉴 하단 위치를 계산한다.

다음 조건을 만족할 때 `window.scrollBy()`를 호출한다.

```text
overflow = menuBottom + 16 - viewportHeight
overflow > 0
```

스크롤 옵션은 `{ top: overflow, behavior: "smooth" }`를 사용한다. effect가 정리될 때 예약된 animation frame을 취소해 닫히거나 언마운트된 컴포넌트가 뒤늦게 스크롤하지 않도록 한다.

## 검증

- 목록이 뷰포트를 벗어나면 계산된 overflow만큼 `smooth` 스크롤을 요청한다.
- 목록이 뷰포트 안에 있으면 스크롤을 요청하지 않는다.
- 드롭다운이 닫혀 있으면 위치 측정이나 페이지 스크롤을 수행하지 않는다.
- 기존 정렬, 선택, 키보드 탐색, 목록 내부 스크롤 동작이 유지되는지 테스트와 빌드로 확인한다.
