# SurveyIntroduce 필수 입력 모달 설계

## 목표

`SurveyIntroduce` 제출 시 현재 페이지 또는 이전 설문에 누락된 필수 입력이 있으면 API를 호출하지 않고 공용 필수 입력 모달을 표시한다. 제출 과정의 API가 HTTP 400을 반환해도 같은 모달을 표시한다.

## 요청 전 검증

1. 제출 중이면 중복 실행을 막는 기존 동작을 유지한다.
2. 닉네임, 자기소개, 공개 프로필 항목 중 하나라도 누락되면 `RequiredFieldsModal`을 열고 즉시 반환한다.
3. 저장된 이전 설문 초안에 누락된 필수 응답이 있어도 페이지를 이동하지 않고 같은 모달을 연다.
4. 누락된 상태에서는 `changeNickname`과 `postSurveys`를 호출하지 않는다.
5. 닉네임 8자 초과와 자기소개 200자 초과는 누락 오류가 아니므로 기존 하단 인라인 오류 문구를 유지한다.

## API 오류 처리

- `changeNickname` 또는 `postSurveys`가 HTTP 400을 반환하면 `RequiredFieldsModal`을 연다.
- HTTP 400에서는 일반 서버 오류 인라인 문구를 표시하지 않는다.
- 400이 아닌 오류는 기존처럼 콘솔에 기록하고 `getSurveyErrorMessage` 결과를 하단에 표시한다.
- 오류 발생 시 사용자 정보 갱신, 설문 초안 삭제, 인증 페이지 이동을 실행하지 않는다.
- 제출 상태는 `finally`에서 항상 해제한다.

## 모달

- 기존 공용 `RequiredFieldsModal`을 사용한다.
- 제목은 `입력 확인`, 본문은 `모든 항목은 필수 입력입니다.`이다.
- 확인 버튼은 없으며 닫기 버튼, 배경 클릭, Escape 키로 닫을 수 있다.

## 테스트

- 현재 페이지 필수 입력 누락 시 모달을 여는 연결을 검사한다.
- 이전 설문 누락 시 페이지 이동 대신 모달을 여는 연결을 검사한다.
- HTTP 400 분기에서 모달을 여는 연결을 검사한다.
- 길이 초과와 400이 아닌 오류의 인라인 메시지 처리가 유지되는지 검사한다.
- 집중 테스트, 린트, 프로덕션 빌드를 실행한다.
