# 로고 자산 적용 디자인

## 목표

`public`에 제공된 서비스 로고와 플랫폼별 아이콘을 각 용도에 맞게 연결한다. 서울시립대학교 로고는 로그인 화면의 배경 워터마크로 유지하고, 율곡 서비스 로고와 혼용하지 않는다.

## 자산 매핑

- `logo.svg`: 로그인 화면 상단의 서비스 로고. 원본 비율을 유지하고 장식이 아닌 브랜드 이미지로서 의미 있는 대체 텍스트를 제공한다.
- `favicon.ico`, `favicon-16.png`, `favicon-32.png`: 브라우저 탭과 북마크 아이콘. ICO를 기본 fallback으로 두고 PNG 크기를 명시한다.
- `apple-touch-icon.png`: iOS 홈 화면 아이콘.
- `icon-192.png`, `icon-512.png`: 설치형 웹 앱용 아이콘. 웹 앱 매니페스트에서 크기와 MIME 타입을 명시한다.
- `uos_logo.svg`: 로그인 화면의 학교 워터마크로 계속 사용한다.

## 화면 및 메타데이터 변경

로그인 헤더의 비어 있는 이미지와 임시 서비스명 텍스트를 `logo.svg` 한 개로 교체한다. 로고는 모바일 헤더에 맞는 고정 높이와 자동 너비를 사용하여 왜곡을 막는다.

`index.html`의 삭제된 `favicon.svg` 참조를 새 파비콘과 Apple touch 아이콘으로 교체한다. `manifest.webmanifest`를 추가하고 192px 및 512px 아이콘을 연결하여 Android 및 설치형 브라우저가 적절한 해상도를 선택하게 한다.

## 프로필 fallback

채팅 사용자 프로필의 fallback은 서비스 로고로 바꾸지 않는다. 서비스 브랜드가 사용자의 얼굴처럼 보이는 혼동을 피하기 위해, 이번 변경에서는 삭제된 `favicon.svg` 의존성만 제거하고 중립적인 CSS 기반 아바타 또는 기존 중립 fallback으로 통일한다. 네트워크 이미지 오류 시에도 깨진 이미지 아이콘이 노출되지 않아야 한다.

## 접근성 및 실패 처리

- 서비스 로고에는 서비스명을 설명하는 대체 텍스트를 제공한다.
- 배경 학교 로고는 장식 요소이므로 빈 대체 텍스트와 `aria-hidden`을 유지한다.
- 모든 로고와 아이콘 URL은 Vite의 `public` 루트 절대 경로를 사용한다.
- 프로필 이미지 로딩 실패는 중립 fallback으로 전환하고 반복 오류를 만들지 않는다.

## 검증

- 정적 검색으로 삭제된 `/favicon.svg`와 `/icons.svg` 참조가 남지 않았는지 확인한다.
- 로그인 화면 소스에서 서비스 로고의 경로, 대체 텍스트, 비율 유지 스타일을 확인한다.
- HTML과 매니페스트의 아이콘 경로 및 크기가 실제 파일과 일치하는지 확인한다.
- 기존 테스트와 프로덕션 빌드를 실행한다.

## 브랜치와 작업 격리

기존 `feature/notify-errors` 작업 폴더의 미커밋 변경을 보존한다. 구현은 `main`에서 분기한 `feature/logo` 브랜치의 별도 worktree에서만 수행한다.
