# 선택형 버튼 피드백 디자인

## 목표

`UserDetails`와 `Surveys` 페이지의 선택형 버튼이 선택 여부와 클릭 반응을 명확히 전달하도록 표현을 통일한다.

## 범위

- 공용 `RadioBtnGroup`
- 설문용 `MultipleBtnGroup`

다음/이전 버튼, 드롭다운 버튼, 소개 항목 추가 버튼은 변경하지 않는다.

## 시각 상태

- 미선택 버튼은 현재의 흰색 배경과 기본 텍스트를 유지한다.
- 선택 버튼도 흰색 배경을 유지한다.
- 선택 버튼의 테두리와 텍스트는 `brand-primary` 색상을 사용한다.
- 선택 버튼의 원래 텍스트 앞에 장식용 체크 문자 `✓`를 추가한다.
- 체크 문자는 스크린 리더가 중복해서 읽지 않도록 `aria-hidden="true"`로 표시한다.
- 기존 `aria-pressed`를 유지해 선택 상태를 접근성 트리에 전달한다.

## 상호작용

- 버튼을 누르는 동안 `scale(0.95)`로 축소한다.
- 누름이 끝나면 transform transition을 통해 원래 크기로 돌아온다.
- 상태 변경 로직, 단일/다중 선택 규칙, 최대 선택 개수는 변경하지 않는다.

## 구현 방식

새 공통 컴포넌트는 만들지 않는다. 두 컴포넌트의 버튼 구조와 크기 차이를 유지하면서 같은 Tailwind 상태 클래스와 동일한 체크 마크업을 각각 적용한다. 이 범위에서는 별도 추상화보다 기존 구조를 유지하는 편이 변경량과 결합도를 낮춘다.

## 검증

- 정적 소스 테스트로 두 컴포넌트가 선택 상태 색상, 체크 표시, 누름 모션, `aria-pressed`를 포함하는지 검증한다.
- 전체 빌드와 린트를 실행해 JSX 및 Tailwind 클래스 변경에 회귀가 없는지 확인한다.
